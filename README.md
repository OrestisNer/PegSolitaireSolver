# PegSolitaireSolver
Java program which solves peg solitaire game


Εργασία 1  Peg Solitaire

Α)Εισαγωγή
Το peg solitaire είναι παιχνίδι ενός παίχτη στο οποίο μετακινεί πασσάλους σε ένα ταμπλό με κενά. Σκοπός του παιχνιδιού είναι να μείνει στο ταμπλό μόνο ένας πάσσαλος. Τότε τελειώνει το παιχνίδι. Περισσότερα: http://en.wikipedia.org/wiki/Peg_solitaire

Β)Πολυπλοκότητα:
Σε ένα ταμπλό με 32 πασσάλους υπάρχουν  577,116,156,815,309,849,672 διαφορετικές ακολουθίες του παιχνιδιού. ( 577+ πεντακισεκατομμύριον πιθανά παιχνίδια) με 40,861,647,040,079,968 λύσεις. (Πηγή: http://www.durangobill.com/Peg33.html)
Ένας αλγόριθμος που ψάχνει «τυφλά» δεν θα δουλέψει γρήγορα ακόμη και σε μια πολύ γρήγορη μηχανή θα πάρει χρόνια να βρει τη λύση.
Σε προτεινόμενες λύσεις υπάρχει ο αλγόριθμος «πρώτα σε βάθος» ο οποίος ψάχνει ένα κλαδί του δέντρου και φεύγει από αυτό μόνο όταν βρεθεί σε αδιέξοδο. Δεν μπορεί να χαρακτηριστεί αποδοτικός για ένα τέτοιο πρόβλημα καθώς δεν λύνει τα προβλήματα σε λογικό χρόνο.
Ένας πιο αποδοτικός αλγόριθμος είναι ο «πρώτα στο καλύτερο» ο οποίος κάνει πληροφορημένη αναζήτηση. Βαθμολογεί με μια ευρετική συνάρτηση τον κάθε κόμβο και πηγαίνει σε αυτόν που πιστεύει ότι είναι πιο κοντά στη λύση.
Ο αλγόριθμος γίνεται περισσότερο αποδοτικός όταν κρατάμε τα ταμπλό που έχουν είδη ελεγχθεί για να μην κάνουμε επαναλαμβανόμενη αναζήτηση. Αυτό υλοποιήθηκε με λίστα κατακερματισμού(HashMap). 
Παρατηρήθηκε ότι για προβλήματα διαφορετικών διαστάσεων και αριθμών πασσάλων χρειάζονται διαφορετικές ευρετικές συναρτήσεις για να φτάσουν στη λύση πιο γρήγορα.  
Γ)Ευρετική Συνάρτηση:
Στον συγκεκριμένο αλγόριθμο η ευρετική συνάρτηση παίρνει ως δεδομένα το μήκος και το πλάτος του ορθογωνίου που συμπεριλαμβάνει όλους τους πασσάλους, το πλήθος των απομονωμένων πασσάλων, το πλήθος των πασσάλων που βρίσκονται στα σύνορα του ταμπλό καθώς και το πλήθος των πασσάλων που βρίσκονται στη κεντρική γραμμή και στη κεντρική στήλη. Ο τρόπος που επιλέχθηκαν αυτά τα δεδομένα είναι κυρίως από παρατήρηση των ταμπλό που δεν μπορούν να γίνουν άλλες κινήσεις γιατί ο σκοπός είναι ο αλγόριθμος να tτα αποφεύγει όσο γίνεται. 
Δ)Αρχείο Εξόδου:
Το αποτέλεσμα που θα γραφτεί στο αρχείο εξόδου είναι ο αριθμός των κινήσεων που χρειάστηκε για να λυθεί , όλες οι κινήσεις , ο χρόνος που χρειάστηκε καθώς και οι κόμβοι που επισκέφθηκε μέχρι να βρει λύση. 

Παρατηρήσεις:
* Οι απληροφόρητοι αλγόριθμοι δεν είναι αποδοτικοί λόγω της πολυπλοκότητας των προβλημάτων.
* Η ευρετική συνάρτηση παίζει καίριο ρόλο στην αποδοτικότητα του αλγορίθμου. Αυτό φαίνεται στους χρόνους και στους κόμβους που επισκέπτονται οι αλγόριθμοι, στα προβλήματα που λύνουν και οι δύο.
* Διαφορετικά προβλήματα χρειάζονται διαφορετικές ευρετικές συναρτήσεις για να λυθούν σε λογικό χρόνο.

Όλα τα ταμπλό (board ) βρίσκονται σε μια κλάση board στον πηγαίο κώδικα.
